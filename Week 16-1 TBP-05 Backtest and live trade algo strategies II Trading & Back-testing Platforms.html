1️⃣ CORE EXAMPLE — "Buy-Low-Sell-High" on SPY (These Numbers Drive Every Section Below)

THE IDEA — mean reversion in 2 numbers

The whole lecture rides on one tiny strategy: an index ETF that fell yesterday tends to bounce tomorrow. So:

if today's close < yesterday's close:

    → BUY all-in (go 100% long)

else:

    → SELL everything (go to cash / short)

It needs only two inputs. We engineer them as daily returns:

yield_yesterday = (close − close.shift(1)) / close.shift(1)
yield_tomorrow  = yield_yesterday.shift(−1)  # what we earn

WORKED EXAMPLE (one day):

Yesterday close = 400, Today close = 396

yield_yesterday = (396−400)/400 = −1.0% → it FELL

Rule: fell → BUY (decision = +1)

Tomorrow it rises to 400: yield_tomorrow = +1.01%

result = decision × yield_tomorrow = +1 × 1.01% = +1.01% ✓

💡 Why "all-in"? Testing fancy position sizes = more knobs = overfitting. If the idea doesn't work in the dumbest 0%/100% form, it won't work fancy either.

THE PROOF — a regression slope, not a hunch

Fit a straight line: does yesterday's move predict tomorrow's?

x = yield_yesterday, y = yield_tomorrow

LinearRegression().fit(x, y) → slope = −0.1012

slope < 0  ⇒  down day → up day = MEAN REVERSION ✓

2️⃣ THE MACHINE-LEARNING VIEW — Slope Sign Tells You Which Strategy You Have

WHAT the slope means

The model IS the strategy. "Buy if it fell" is just a linear regression with one feature. The slope's sign classifies the market:

SPY = −0.10 → reversion → "buy the dip" is correct here.

WHY a screener helps

Run the regression over many tickers and sort by slope. The number tells you where the edge lives — you don't guess.

WORKED: SPY −0.095 beats GOOG −0.022 → SPY reverts ~4× harder, so it's the better candidate for this rule.

💡 Jim Simons (Renaissance): "The most common tool we use is linear regression." The simplest model, used relentlessly, beats clever ones that overfit.

HOW to improve it

If a 1-feature model is too weak (underfit), add a feature — e.g. MACD (a momentum oscillator) as a second x-column:

X = df[["yield_yesterday", "macd"]]
y = df["yield_tomorrow"]
LinearRegression().fit(X, y)

⚠ Each feature is a knob. More features can fit the past better and the future worse. Add them only if you have lots of data (see §4).

4️⃣ WHY A GREAT BACKTEST IS A LIE — The Two Traps That Kill It

TRAP 1 — Transaction costs

The strategy flips position almost every day. Every flip pays the spread + commission. Charge a tiny cost per flip and watch it compound to ruin:

Cost / flipFinal $1 →Return
0.00%×19.03+1803%
0.10%×2.4+140%
0.50%×0.12−88%

HOW THE EDGE GETS EATEN (gross → minus cost → net):

Cost / flipGross (no cost)Net (after cost)Edge left
0.00%×19.03×19.03100%
0.10%×19.03×2.4~13%
0.50%×19.03×0.12gone

Same gross curve every row → a tiny per-flip cost compounds over ~thousands of flips and erases it.

💡 Rule: high-frequency strategies die first to costs. Always subtract realistic costs before believing a curve.

TRAP 2 — Over-fitting (the honest test)

We chose 1.1% by peeking at the whole history. The fair test: tune on the first half, then trade the second half you never saw.

TestStrategyBuy & Hold
In-sample (peeked)×19.03×4.53
Out-of-sample (fair)×1.07×3.84

"Torture the data long enough and it confesses to anything."

MEASURE IT — over/under-fit = data per parameter (not the train/test gap)

data per param = rows ÷ (# tuned knobs)

WORKED (our model):

knobs = today close, yesterday close, tuned rate = 3

5000 rows ÷ 3 ≈ 1666 per param

Healthy minimum for price models ≈ 1000 → we're slightly underfit (safe-ish), not overfit.

⚠ Overfit ≠ "big train/test gap." That gap is a symptom. The cause is too many knobs for too little data. Fewer knobs + more data = safer.

5️⃣ WHEN DO WE TRADE? — The 3:59 PM Trick

The chicken-and-egg problem

Our rule needs today's close to decide — but the market closes at 4:00 PM and then it's too late to trade today. Fix: decide at 3:59 PM, when the spot price ≈ the close.

Place a market order for an instant fill at 3:59.

PROVE it's safe — a t-test

Is the 3:59 price really close enough to the 4:00 close? Don't assume — test it:

diff = price(3:59) − close(4:00)

H₀: mean(diff) = 0

scipy.stats.ttest_1samp(diff, 0)

p > 0.05 → can't reject → 3:59 ≈ close ✓

💡 This is the quant habit: every "obviously fine" assumption gets a statistical check before real money rides on it.

8️⃣ A SECOND STRATEGY FROM A PAPER — "VIX Predicts Returns"

The rule (Giot, 2005)

VIX = the market's "fear gauge." Using a rolling 2-year window of VIX percentiles:

VIX todayAction (1 day)
highest ever / top 2 boxesLONG the index
lowest ever / bottom 2 boxesSHORT the index
anything in betweenFLAT (cash)

Extreme fear → market tends to bounce (go long); extreme calm → complacency, fade it (go short).

The reusable workflow (any paper → robot)

1. Read paper → restate the rule plainly

2. Get the data (here: a 2nd instrument, VIX,

   via superSymbol(secType='IND', symbol='VIX'))

3. schedule_function at market close

4. compute percentile bands → decide

5. order_target() → backtest → forward-test

Key numbers (real SPY)

  • Regression slope = −0.1012 (revert)
  • Threshold 0% → ×5.24
  • Threshold 1.1% → ×19.03 (in-sample)
  • Buy & hold → ×4.53
  • 0.5% cost/flip → −88%
  • Out-of-sample → ×1.07 vs ×3.84
  • Data/param ≈ 1666 (need ≥1000)

Idea → live robot (9 steps)

  1. Idea / observation
  2. Get data
  3. Model it (regression)
  4. Graph it
  5. Code the rule
  6. Backtest + overfit check
  7. Paper-trade
  8. Go live small
  9. Track & improve

Same path scales into a fund: track record → outside accounts → multi-account.

Top gotchas

  • No costs = fantasy curve.
  • Peeking to pick a knob = overfit.
  • Overfit = data-per-param, not the train/test gap.
  • Judge by Sharpe, then forward-test.
  • Run reversion on an index, not hot stocks.
  • If it can't earn in the model, it never will live.
Glossary: Sharpe = return ÷ risk · MACD = momentum oscillator · slippage = price moves before fill · t-test = "is this difference really zero?" · superSymbol = define a non-stock instrument (e.g. VIX).
