⑥ The code skeleton

from ibapi.client import EClient
from ibapi.wrapper import EWrapper
from threading import Thread

class Strategy(EClient, EWrapper):
    def __init__(self):
        EClient.__init__(self, self)   # 'double self' wires both parents
    def currentTime(self, t):          # CALLBACK (server calls it)
        print(datetime.fromtimestamp(t))

app = Strategy()
app.connect("127.0.0.1", 7496, clientId=1)
Thread(target=app.run, daemon=True).start()   # listener loop
app.reqCurrentTime()                 # -> currentTime() fires
connect() argMeaning
hostIP of TWS machine (localhost if same box)
portset in IB API settings (e.g. 7496)
clientIdany int; up to 32 clients/TWS; identifies the strategy

⑦ Why threading?

app.run() starts an infinite listener loop — without it you get no responses; with it, it blocks all code below. So run it on a daemon Thread while the main thread keeps sending requests.

⚠️ No callback defined → response is lost. The request still goes out & the server still answers, but your program can't handle it.

📊 The architecture & the sync-vs-async timeline

Top: request flows out via EClient, response returns via EWrapper callback · Bottom: SYNC idles between requests, ASYNC overlaps them (verified in the notebook: 0.15s vs 0.06s)

⏱️ 30-second glossary & gotchas

API — software medium between client & server; structured request/response.
EClient / EWrapper — request side / callback side. Inherit both into Strategy, object = app.
app.run() — infinite listen loop; run on a daemon thread.
"Errors" on startup = data-farm connection info, not errors. Ignore.
connect comes from EClient (via Strategy's inheritance).
Use Spyder/VS Code, NOT Jupyter, for live trading (Jupyter = research tool, no async loop).
