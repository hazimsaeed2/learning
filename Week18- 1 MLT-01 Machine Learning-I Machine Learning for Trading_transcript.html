TRAINING vs TESTING — The Kid Analogy

TRAINING = showing kid fruits + labels
→ Kid explores features (weight, color, taste)
→ We label: "this is apple", "this is orange"
→ Kid forms ASSOCIATION: features → labels

TESTING = showing kid NEW fruit, no labels
→ Kid uses learned association to PREDICT
→ We check: was the kid right?

CRITICAL: Test data must be data the model
has NEVER seen before. Otherwise you're just
measuring memorization, not learning.

4️⃣ LOGISTIC REGRESSION — From Linear to Sigmoid: The First Classification Algorithm

THE PROBLEM WITH LINEAR REGRESSION ON BINARY DATA

Linear Regression: Y = α × RSI + β

Works great when Y is numerical:
X = RSI (numerical) → Y = Asset Value (numerical)
Line of best fit represents data well ✅

FAILS when Y is binary (0 or 1):
X = RSI (numerical) → Y = Direction (0 or 1)
No straight line can fit binary scatter plot ❌
Sum of squared errors always very high
No matter what line you draw → poor fit

THE SOLUTION: SIGMOID / LOGIT FUNCTION

Step 1: Z = α × feature + β (linear function)
Step 2: σ(Z) = 1 / (1 + e−Z) (sigmoid)

When you pass the linear function through
the sigmoid, the straight line transforms
into an S-shaped curve that fits binary data!

Output: probabilities between 0% and 100%

If σ(Z) > 50% → predict class 1 (up)
If σ(Z) ≤ 50% → predict class 0 (not up)

The 50% threshold is default but ADJUSTABLE
(50% seldom works in practice → try 60%, 70%)

WORKED EXAMPLE — Probability → Prediction

ObsP(class=0)P(class=1)PredictionWhy
#137%62%1 (up)62% > 50%
#246%53%1 (up)53% > 50%
#356%43%0 (not up)43% < 50%

model.predict_proba() returns probabilities for each class. model.predict() returns the final 0/1 based on threshold.

KEY FACTS

Despite the nameLogistic Regression is a CLASSIFICATION algorithm (not regression!)
Binary classificationPrimary use: two classes (0/1, up/down)
Multi-class extensionMultinomial / Softmax logistic regression
Multiple featuresZ = α₁×f₁ + α₂×f₂ + ... + αₙ×fₙ + β

⚠️ GOTCHA: Regression IS a machine learning algorithm (one and the same — you already learned it in statistics classes). That's why the ML lectures focus on classification and clustering, not regression. If you've done linear regression, you've already done supervised ML.

5️⃣ PYTHON MODEL BUILDING — Complete Code: Target, Features, Split, Scale, Train, Predict

STEP 1-3: TARGET & FEATURES

import pandas as pd               # Data analysis
import numpy as np                 # Vectorization
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

# ── Read data ──────────────────────────────
data = pd.read_csv('tesla.csv')   # OHLCV data
df = data.copy()                   # Preserve original

# ── Create target variable ─────────────────
df['returns'] = np.log(            # Log returns
    df['Close']).diff()
df['target'] = np.where(           # Label: up or not
    df['returns'].shift(-1) > 0,   # shift(-1) = NEXT day
    1,                              # 1 = stock went up
    0)                              # 0 = stock did NOT go up
# shift(-1) is NOT look-ahead bias here
# We're PREPARING data, not training yet

# ── Feature extraction ─────────────────────
# Volatility: rolling std of close price
for period in range(5, 21, 5):     # 5, 10, 15, 20
    df[f'vol_{period}'] = (
        df['Close'].rolling(period).std())

# Trend: rolling mean of close price
for period in range(10, 31, 5):    # 10, 15, 20, 25, 30
    df[f'trend_{period}'] = (
        df['Close'].rolling(period).mean())

# Momentum: cumulative pct returns
for period in [3, 6, 9]:
    df[f'mom_{period}'] = (
        df['Close'].pct_change()
        .rolling(period).sum())

# Volume: rolling mean
df['vol_MA'] = df['Volume'].rolling(10).mean()

# Intraday movement
df['intraday'] = abs(df['Close'] - df['Open'])

STEP 4-9: SPLIT, SCALE, TRAIN, PREDICT

# ── Drop NaN (from rolling features) ──────
df.dropna(inplace=True)            # ~1,486 rows remain

# ── Define X and y ─────────────────────────
features = ['vol_5', 'vol_10',     # All feature columns
    'vol_15', 'vol_20',            # ... list all features
    'trend_10', 'mom_3',
    'vol_MA', 'intraday']          # etc.
X = df[features]                   # Feature MATRIX (capital X)
y = df['target']                   # Target VECTOR (lowercase y)

# ── Split: train/test ─────────────────────
X_train, X_test, y_train, y_test = (
    train_test_split(
        X, y,
        test_size=0.25,            # 25% test, 75% train
        shuffle=False))            # ⚠️ CRITICAL for time series!

# ── Scale: StandardScaler ─────────────────
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(  # fit + transform
    X_train)                        # on TRAINING data only!
X_test_scaled = scaler.transform(   # transform ONLY
    X_test)                         # using TRAIN mean/std!

# ── Train model ───────────────────────────
model = LogisticRegression(
    random_state=100)               # Seed for reproducibility
model.fit(X_train_scaled, y_train)  # Learn associations

# ── Predict & evaluate ─────────────────────
y_pred_train = model.predict(X_train_scaled)
y_pred_test = model.predict(X_test_scaled)
probs = model.predict_proba(X_test_scaled)

train_acc = (y_pred_train == y_train).mean()
test_acc = (y_pred_test == y_test).mean()
# train_acc ≈ 0.56 (56%)
# test_acc  ≈ 0.53 (53%)

Feature Selection = choosing from existing data. Feature Extraction = creating NEW features from existing data. Both are critical.

💡 THE DIFFERENTIATOR: The lecture emphasizes that feature design is where your natural intelligence enters the process. Create a hypothesis first, then build features to test it, rather than throwing every possible feature into the model.

DATE/TIME AS FEATURES?

Usually ignored, but some practitioners add minute, hour, day-of-week as columns to capture anomalies (e.g., mutual funds roll over positions in last week of expiry → predictable buying patterns).

STANDARD SCALING — Why & How

PROBLEM: volume_MA ≈ 4,000,000
momentum_3 ≈ 0.05
→ Algorithm thinks volume is "more important"
because bigger numbers → biased learning!

SOLUTION: Z-score scaling
Z = (value − mean) / std_dev
After: ALL features have mean=0, std=1

PRESERVES: distribution shape, frequencies
CHANGES: only the scale (range of values)

THREE CRITICAL RULES TO AVOID LOOK-AHEAD BIAS

#RuleIf You Break It
1shuffle=False in train_test_splitYou train on 2024, test on 2014 → cheating
2fit_transform on train, transform only on testUsing future mean/std to scale past data
3Never use test data info during trainingInflated accuracy that won't hold in live trading

⚠️ MOST COMMON MISTAKE: Using scaler.fit_transform(X_test) on test data. This computes mean/std FROM the test data, leaking future information. Always use scaler.transform(X_test) (no fit!) so test data is scaled using TRAINING mean/std only.

7️⃣ CLUSTERING — Unsupervised Learning: Group Similar Things Without Labels

HOW IT WORKS

1. Collect features only (NO labels/targets)
Example: 50 companies → returns & volatility

2. Tell algorithm: "make 3 clusters"
(YOU specify the number of clusters)

3. Algorithm groups similar observations
Output: company → cluster_1, cluster_2, etc.

4. YOU provide context to each cluster
Dig deeper: WHY are these companies similar?
Algorithm might cluster by gender, sector,
or some hidden pattern not in the features!

🎯 KEY TAKEAWAYS

1. ML has three tasks: Regression (predict numbers), Classification (predict classes), Clustering (group similar things). Regression and classification are supervised (need labels). Clustering is unsupervised (features only, no labels).

2. Logistic Regression = classification, not regression. It passes a linear function through a sigmoid to output probabilities (0-100%). Default threshold is 50% but should be tuned. Despite its name, it classifies binary outcomes like up/down.

3. Features make or break your model. Don't kitchen-sink. Build a hypothesis first, then create features to test it. This is where YOUR natural intelligence differentiates you from every other analyst using the same data and algorithms.

4. Three rules to avoid look-ahead bias: (1) shuffle=False for time series, (2) fit_transform on TRAIN data only, transform-only on TEST, (3) never use test data statistics during training. Break any rule → inflated accuracy that fails in live trading.

THE MASTER WORKFLOW: Data → Target → Features → Drop NaN → Split (shuffle=False) → Scale (fit on train only) → Train → Predict → Evaluate. This is the core supervised-learning workflow demonstrated in the lecture.
