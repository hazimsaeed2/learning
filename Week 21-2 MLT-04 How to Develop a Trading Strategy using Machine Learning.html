The Big Idea — Trust the Process, Not the Number

Machine learning in trading is 90% research discipline, 10% model. The hardest skill isn't picking an algorithm — it's not leaking the future into the past. A leaked backtest looks brilliant and dies live.

Every result must be measured against a benchmark: a dummy classifier (for the model) and buy-and-hold (for the strategy). On XLF, 54% of days are already "up" — so 54% accuracy means the model learned nothing.

Negative, honest results are the normal output of real research — and infinitely more valuable than a leaked fantasy. Our pipeline below ends with a model that fails its honest holdout (test-set MCC −0.006). That's a success: it told the truth.

Module 1 — Benchmarks, Splits, Walk-Forward, MCC

Temporal splits — never random

SplitUsed forRule

Validation (20%)hyper-parameter tuningtune here, freely

Time has order — a random split lets the model peek at the future. Always split in chronological order.

Walk-forward + embargo

Train on the past, predict the next block, step forward — exactly like production. A small embargo gap at each boundary purges the temporally-adjacent rows that would leak look-ahead info.

for k in 1..n_splits:
  tr_end   = fold*k
  te_start = tr_end + embargo   # gap!
  fit(X[:tr_end]); predict(X[te_start:te_end])

Why accuracy lies — use MCC

With XLF's 54% "up" days, a model that always predicts "up" scores 54% accuracy while knowing nothing. The Matthews Correlation Coefficient (−1…+1, 0 = random) balances TP/TN/FP/FN and survives class imbalance.

MCC = (TP·TN − FP·FN) / √((TP+FP)(TP+FN)(TN+FP)(TN+FN))

The Three Leakage Anti-Patterns — the Heart of the Lecture

The three canonical traps are the shuffle split, the unshifted feature, and K-fold — each inflates MCC upward, a fantasy edge that evaporates live. Reproduced below on the same real 88-feature XLF data with the same LightGBM model, versus the honest walk-forward baseline.

MethodMCCInflation vs honestWhy it leaks
Honest walk-forward+0.035— baseline —trains past → predicts future, with embargo
1 · Random shuffle split+0.559+0.525train_test_split(shuffle=True) scatters future rows into train
2 · K-fold (no shuffle)+0.046+0.011future folds train a model judged on past folds
3 · K-fold + shuffle+0.582+0.548worst of both — only valid for IID data, never time series

Every leaky method scores above the honest +0.035 — and shuffling explodes it to +0.56, a spectacular fantasy edge. The bigger the inflation, the more dangerous the false confidence.

Module 4 — Tune on Validation, Test Once, Backtest

① Tune (smart search)

Hyper-parameters are model knobs, not features (depth, n_iter, learning rate). Grid search is too slow → smart random/Bayesian sampling on the validation set.

valid MCC = +0.019
(SHAP-18, depth 10, 50 iters)

Validation already ≈ 0 → a careful researcher would stop and rethink here.

② Test (one shot)

Retrain on train+val, predict the holdout exactly once. Never retune or peek.

train +0.066
valid +0.019
test −0.006

70% degradation train→valid, then negative on the untouched test. Verdict: ❌ FAIL, not production-ready. The system working, not failing.

③ Backtest (event-driven)

Vectorized holdout backtest, long when P(up) > 0.5, over the 2019+ test window — even before realistic spreads and fills.

return +5.6%
Sharpe 0.15
max DD −20.4% · 91 trades

Barely positive, but Sharpe 0.15 ≪ the 1.5 a deployable strategy needs. Bad results are information.

The seven rules to carry away

  1. Benchmark first — dummy classifier + buy-and-hold set the floor; beat it or stop.
  2. Split temporally — train / validation / test, never random.
  3. Walk-forward with an embargo — mimic production, purge boundary leakage.
  4. Score with MCC, not accuracy — imbalance makes accuracy lie.
  5. Avoid the anti-patterns — shuffle, no-shift(1), and k-fold all inflate.
  6. Prune with SHAP — 88 → 18 features, same signal, far easier to maintain.
  7. Tune on validation, test once, then backtest event-driven with costs.
