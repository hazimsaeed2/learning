Python Basics for Quants

Week 6-3 · PBQ-02 — your second Python lecture, read as a continuous lesson. From data structures and loops to the two libraries every back-test rests on: NumPy and pandas.

The first half of this lecture consolidates the Python you already met — data types, loops, functions — because these are the grammar of everything that follows. The second half introduces the real reason quants use Python: NumPy for fast numerical computing and pandas for working with tabular market data. By the end you can read a CSV of stock prices into a table, add a column, summarise it, and see the story the numbers tell. Throughout we use the actual TCS.NS.csv daily file that ships with the class.

1. Data types and structures — and why syntax matters

Python reads your intent from syntax alone. 2 is an integer, 2.0 a float, "TCS" a string, and a comparison like 3 > 2 a boolean. Those four hold single values. For collections you have four structures, each with a distinct job:

StructureSyntaxDefining trait
list[ ]ordered, mutable, can mix types
tuple( )ordered, immutable (can't be changed)
dictionary{key: value}key→value pairs, gives context
set{ }only unique values, order irrelevant

Suppose you want to store a quote: symbol "TCS", price 1175.72, volume 1520352. A list or tuple works, but a newcomer reading the code has no idea what each value means. A dictionary — {"symbol":"TCS", "last_price":1175.72, "volume":1520352} — is self-documenting; you fetch the price with quote["last_price"]. The set's two quirks are worth fixing in memory: it discards duplicates ({"a","u","a"} becomes {"a","u"}) and ignores order, so {"a","u"} == {"u","a"} is True whereas the same comparison on lists is False.

2. Methods versus attributes

Every structure carries its own methods (actions you call) and attributes (values you read). A list has .append() — a method, so it needs parentheses — but a tuple does not, because a tuple is immutable. Try ("a","b").append("c") and Python replies "'tuple' object has no attribute append."

Two ways to get help fast: type a name followed by a dot and press Tab to see every method and attribute available; or put a ? after a function (range?) to read its docstring — the built-in description of what it does.

3. Control flow: loops and branching

Logic flows through loops and conditionals. The workhorse is the for loop paired with if/elif/else. Imagine a simple rule: buy the market when quarterly GDP growth exceeds 7%, hold at exactly 7%, sell below. Iterating over [7.2, 7, 8, 6.6] produces buy, hold, buy, sell — and you collect the decisions by .append()-ing each into a list.

GDP %ConditionSignal
7.2> 7buy
7.0= 7 (the else)hold
8.0> 7buy
6.6< 7sell

Three things make the loop work: the colon after each condition is mandatory syntax, and indentation is meaningful — the indented block belongs to the loop or branch above it; mis-indent and you get an IndentationError. The same decision can be written as a one-line if (a ternary): "buy" if g > 7 else ("sell" if g < 7 else "hold") — shorter, though not faster.

List comprehensions

A list comprehension compresses a whole loop into one line: [expression for item in list]. The squares of [2,3,4,5] are [i*i for i in [2,3,4,5]], and the entire GDP-signal loop collapses to a single comprehension. They are also handy for transforming prices — [np.log(p) for p in prices] gives log prices in one line.

While loops, break and continue

A while loop runs as long as its condition stays true; you must update the counter inside the loop or it never ends (an infinite loop that hangs the kernel). The instructor's analogy is a trading session: while the clock is between 9:15 and 3:30 keep looking for signals, and set the session flag to false at 3:15 to stop taking new trades. Two keywords steer any loop: break exits the loop entirely, while continue skips only the current iteration and moves to the next value.

4. Functions

A function is a reusable machine: ingredients go in, a result comes out. You have already used built-in ones (len, max), methods (.append), and library functions (np.log). Your own are defined with def, take inputs, and hand back a value with return:

def f(x, y, z):
    k = x**2 + y**2 - z
    return k          # f(2,3,1) -> 12
Printing is not returning. print only displays a value on screen; return sends it out of the function so you can store it and keep working with it. In the bread-factory analogy, printing hands you a photo of the bread; returning hands you the actual loaf.

For very simple logic there is the lambda — an anonymous one-line function, e.g. lambda x: "negative" if x < 0 else "non negative". It isn't faster, but it isn't saved to a named variable, which saves a little memory. Beginners can safely ignore it; the standard def handles every case.

5. Libraries, modules and packages

Decades of code for every field has been bundled into libraries and shared open-source. Quants lean on a handful: NumPy for numbers, SciPy for scientific computing, matplotlib/seaborn for visualisation, scikit-learn for machine learning. Some libraries ship built-in (like os); others you install with pip install first. The os library manages the operating system — os.getcwd() shows your current working directory (where files are read and saved from), os.chdir() changes it, and dir(os) lists everything inside.

6. NumPy — the numeric workhorse

Plain Python lists are not built for maths. Add two lists with + and Python concatenates them ([1,2]+[6,7] → [1,2,6,7]), not what you want. Convert them to NumPy arrays with np.array() and + becomes true element-wise addition ([7,9]) — a vectorized operation. The same applies to multiplication, division, powers and more, all done in a single batch.

Why it's fast: NumPy runs its operations in C, a low-level compiled language. You write easy Python but execute at C speed. Timing max() over a thousand numbers, a list takes roughly 7.5 microseconds versus about 1.5 for an array — around five times faster. On HFT-scale data that gap compounds enormously.

There are several ways to build arrays: np.array from a list; np.linspace(1,10,5) for five equally spaced values including the endpoint (1, 3.25, 5.5, 7.75, 10); np.arange(0,4,0.5) for a stepped range with the endpoint excluded; and np.zeros(5) / np.ones(5) for pre-filled arrays. Every array carries three key attributes: .ndim (number of dimensions), .shape (rows, columns), and .dtype (int64, float64, …).

Random samples and the seed

Monte-Carlo methods — for value-at-risk, derivative pricing — need random draws, and NumPy's np.random sub-library samples from many distributions (normal, uniform, integers). Draw 10,000 from a standard normal and the sample mean sits near 0 and the standard deviation near 1, as expected. But these are pseudo-random: an algorithm produces them from a starting value called the seed. Fix the seed with np.random.seed(17) and you get the exact same "random" numbers every run.

Why seeding matters: you build an analysis on random draws, present it the next day, re-run the code — and without a seed your numbers have changed and your story falls apart. The seed makes results reproducible. The number 17 isn't special; any fixed value works.

One more essential is np.where, which behaves like Excel's IF: np.where(x < 6, "buy", "sell") returns "buy" where the condition holds and "sell" elsewhere. Given only a condition, it instead returns the locations (indices) where the condition is true.

7. pandas — Series and DataFrame

pandas (from "panel data") is built on top of NumPy and is what quants actually use day to day. It offers two structures: a Series (one-dimensional, like a single labelled column) and a DataFrame (two-dimensional, a spreadsheet with rows and columns). A Series improves on a bare array by adding an index — store weekly returns indexed by Mon–Fri and you can ask returns["Thu"] directly, much like a dictionary. A DataFrame has three parts: an index, columns, and the values (stored internally as a NumPy array — which is why pandas and NumPy interoperate so smoothly).

Reading the real TCS data

You read a CSV with pd.read_csv. For time-series data three arguments matter: index_col="Date" makes the date the row label (so you can slice by time), parse_dates=True tells pandas the column is a date rather than text, and dayfirst=True resolves the dd/mm/yyyy format (so 24/11 is the 24th, not an impossible month). The TCS file loads as 1233 daily rows with Open, High, Low, Close, Adj Close, Volume and a Direction column.

From here the everyday moves: df.head()/df.tail() peek at the ends; df["High"] returns a one-dimensional Series while df[["High","Low"]] (note the list inside) returns a two-dimensional DataFrame; and you create a new column with simple vectorized maths.

Plotting df["Close"] and df["Range"] (pandas uses matplotlib behind the scenes) tells the story instantly: a huge spike in the daily range in March 2020 — the COVID crash — and a generally higher volatility regime afterwards. Finally, df.drop(["Adj Close"], axis=1) removes a column, but returns a copy; to change the original you reassign it or pass inplace=True. And df.describe() prints count, mean, standard deviation, min, max and quartiles for every numeric column at once.

Key takeaways

  • Syntax sets the type and chooses the structure: list (mutable), tuple (immutable), dict (context), set (unique, unordered).
  • Method vs attribute: .append() is a called method; tuples lack it. Use .+Tab and ? for help.
  • Control flow: for/while loops with if/elif/else (mind the colon and indentation); break exits, continue skips; list comprehensions compress loops into one line.
  • Functions return (not print) reusable results; lambda is the one-line anonymous form.
  • NumPy gives fast, vectorized arrays (array, linspace, arange, zeros/ones), reproducible randoms via a seed, the ndim/shape/dtype attributes, and np.where (Excel-style IF).
  • pandas (Series, DataFrame) builds on NumPy: read_csv with index_col/parse_dates/dayfirst, column selection, new columns, describe, and drop — and on real TCS data the daily range screams "COVID" in March 2020.
