4

Adapter & normalizer

Each exchange speaks its own binary language. The adapter knows the byte layout; the normalizer unifies formats.

fmt = ">Iffff"  # id, bid, ask, bq, aq
struct.unpack(fmt, raw)
# -> AAPL bid 270.10 ask 270.14
De-cumulate depth: Exchange B sends cumulative qty [700, 1900] → level qty [700, 1200] so both exchanges share one order-book structure.

⚠ Why FIX is slow: it's text (must parse) and bloaty — price "265.01" = 6 bytes vs a 4-byte float. Popular exchanges use native binary; FIX adds a converter hop = latency.

Dispersion & implied correlation

Index variance = component variances + pairwise correlation. Solve for the constant ρ the market prices, then trade index-vol vs stock-vol.

σI² = Σ wᵢ²σᵢ² + ρ·Σi≠j wᵢwⱼσᵢσⱼ

Simulator + data replay

Never go straight live. Run the strategy on an exchange simulator fed by replayed data (flat / volatile / gap days). Fill probability for an in-spread order:

order vs 99/100 bookP(fill)
@ 99.00 (at bid)0.05
@ 99.50 (mid)0.50
@ 100.00 (at ask)0.99
Good simulators are proprietary — modelling iceberg orders, queue position & variable exchange latency needs deep microstructure skill.

Additive details: exchange side + queues

The exchange side is a distributed system: gateways route to matching engines, while market-data, recovery, and drop-copy streams support the trading flow.

Gateway
Receives member sessions, applies checks, and routes each symbol to its matching engine.
Message-rate queue
Holds orders once the purchased orders-per-second quota is used.
Ack queue
Blocks cancel/replace until exchange order ID and timestamp are known.
