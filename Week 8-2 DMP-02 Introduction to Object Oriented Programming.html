Introduction to Object-Oriented Programming

Instructor: Ashutosh Dave — a new paradigm: blueprints (classes) & specific things (objects).

THE PROBLEM
Sequential / functional code gets messy & un-reusable when you run many strategies.
YOUR GOAL
Bundle data + functions under one umbrella so each back-test is a single clean object.
THE EDGE
Class = blueprint, written once; objects = specific runs, created in one line; reuse via inheritance.
CORE EXAMPLE running through this sheet → a Student class (concepts) that grows into a BacktestingCrossover class (real NIFTY data, 962 bars).

1 · Dunders — magic behind operators

A dunder = "double underscore" method. Operators & built-ins secretly call them.

2 + 2          # 4
(2).__add__(2) # 4  ← same thing
len([1,2,3])   # 3
[1,2,3].__len__()  # 3
np.__version__ # '2.3.5'

The one that matters most for OOP: __init__.

2 · Class vs object

  • Class = blueprint / general idea (a "car": 4 tyres, body, engine).
  • Object / instance = a specific one (a blue Mercedes).
  • In Python everything is an object: 4.3 is an object of class float.
type(4.3)        # <class 'float'>
(4.3).is_integer()  # False
(4.0).is_integer()  # True

3 · Class attribute & @classmethod

class Student:
    goal = "to get educated"   # class attr (shared)

    @classmethod               # decorator links to class
    def update_goal(cls, new): # cls = the class
        cls.goal = new         # changes for EVERYONE

Update via the class method → every student's goal changes. Shared state.

4 · __init__, self & instance attrs

class Student:
    def __init__(self, name, age): # runs on creation
        self.name = name           # instance attr
        self.age  = age            # (per object)

john = Student("John Paul", 19)
john.name   # 'John Paul'

self = the object being built. Per-object data lives here.

6 · Inheritance — parent → child

class SportyStudent(Student):   # inherits ALL
    goal = "...win some trophies"  # override 1 thing

dave = SportyStudent("Dave", 17)
dave.update_age(18)             # inherited!
isinstance(dave, Student)       # True (via parent)
issubclass(SportyStudent, Student) # True

Child = best of both worlds: inherits everything, then tweaks a little (like pandas built on numpy).

7 · super() & the MRO

class Child(ParentOne, ParentTwo):
    def __init__(self, grade, age):
        self.grade = grade
        super().__init__(age)   # run parent too

Child.__mro__   # Child→ParentOne→ParentTwo→object

MRO = method resolution order = family tree, always ends in object. Multiple inheritance follows it.

dave.__dict__ → instance attrs; .__mro__ → ancestors.

💡 Tips
  • Recognise a method by its first arg: cls → class method, self → instance, neither → static.
  • Parent class must be defined before the child inherits it.
  • If __init__ requires args, you must pass them: Student("Mary", 20).
⚠ Traps
  • @classmethod is required for class methods — omit it and you get an error.
  • Changing a class attribute via a class method changes it for all instances (shared state).
  • There's no real private in Python — only a leading-underscore convention.

30-second glossary

dunder — __x__ method behind operators (__add__, __len__, __init__).
class — blueprint defined with class Name:.
object / instance — a specific thing built from a class.
class attribute — shared value in the class body.
instance attribute — per-object value set on self in __init__.
self — the object inside its own methods.
cls — the class inside a class method.
inheritance — child class reuses + overrides a parent.
super() — call the parent's method/constructor.
MRO — method resolution order; the ancestry, ends in object.
isinstance / issubclass — test object↔class / child↔parent.
@staticmethod — plain helper grouped under a class.
