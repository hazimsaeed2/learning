2️⃣ WHY SIMPLE TRADING RULES FAIL — Regime Shifts Break Static Strategies

WHAT: Static Rules Can't Adapt

WHY it matters: A simple Moving Average (MA) crossover on SPY (S&P 500 ETF) from 2019–2025 returned only ~1%, while buy-and-hold returned ~200%. The strategy collapsed during COVID, rate hikes, and tariff wars.

import yfinance as yf                     # Yahoo Finance
import numpy as np                         # Math operations

data = yf.download("SPY",                 # S&P 500 ETF
    start="2019-01-01", end="2026-01-01",  # 7 years of data
    multi_level_index=False)               # Clean columns
# Output: ~1800 rows × 5 columns

data['returns'] = data['Close'].pct_change()  # Daily returns

# Signal: +1 if SMA > LMA (trending up), else -1
data['signal'] = np.where(
    data['SMA'] > data['LMA'], 1, -1)

data['strategy_returns'] = (
    data['signal'].shift(1) * data['returns'])

# Result: ~1% total return (buy-and-hold: ~200%)

WHY: Regime Shifts Explained

WHAT is a regime shift? A fundamental change in how markets operate — COVID, wars, rate hikes, tariff wars.

Events that caused regime shifts (2019–2025):

• 2020: COVID-19 pandemic → massive crash, then rapid recovery

• 2022: Fed rate hikes → long-term momentum strategies hammered

• 2022–2023: Ukraine-Russia war, Middle East conflicts

• 2024–2025: US tariff wars → sudden sector rotations

GOTCHA: During high-volatility periods, markets are very choppy and tend to trend downward. MA crossovers generate too many false signals in these conditions.

⚠️ WARNING — Overfitting to calm markets: A strategy that works well in 2019 (low volatility) will collapse spectacularly in 2020 (COVID crash). Always test across multiple regimes.

💡 THE MOTIVATION: Statistical models like ARIMA adapt because they retrain on recent data each period. Instead of fixed rules ("buy when SMA > LMA"), they build forecasting models that adjust as markets change.

3️⃣ STATIONARITY — The Foundation of Time Series Modeling

WHAT: Stationarity = Stability Over Time

WHY: AR and ARIMA models assume the data they work with is stationary. If it's not, forecasts are unreliable.

A process is stationary if:

1. The mean stays roughly constant over time

2. The standard deviation (volatility) stays roughly constant

3. The autocorrelation structure doesn't change

KEY RULE:

• Prices are almost NEVER stationary (they trend up/down)

• Returns are almost ALWAYS stationary (they hover around zero)

→ We model returns, not prices (or use d=1 to difference prices automatically)

WORKED EXAMPLE (using Core Data):

Prices: 50.00, 51.50, 49.80, 52.20, 51.00 → trending, mean shifts

Returns: +3.00%, −3.30%, +4.82%, −2.30% → hovering around 0%

The mean of returns ≈ +0.56%. The mean of prices keeps changing. That's the difference.

HOW: The ADF Test (Augmented Dickey-Fuller)

WHAT: A statistical test that checks whether a series has a "unit root" — a fancy way of asking: is it stationary?

from statsmodels.tsa.stattools import adfuller  # Import ADF test

# Test prices — expect NON-stationary
result_price = adfuller(data['Close'].dropna())
print(f"Prices p-value: {result_price[1]:.4f}")
# Output: p-value = 0.6523  → > 0.05 → NON-stationary ✗

# Test returns — expect stationary
result_returns = adfuller(data['returns'].dropna())
print(f"Returns p-value: {result_returns[1]:.4f}")
# Output: p-value = 0.0000  → < 0.05 → STATIONARY ✓

DECISION RULE:

p-value < 0.05STATIONARY ✓Safe to model
p-value ≥ 0.05NON-STATIONARY ✗Must transform (diff or returns)

Null hypothesis: Series is non-stationary. If p < 0.05, we reject the null → series IS stationary.

WHAT is differencing? Instead of modeling prices [50, 51.50, 49.80, 52.20], model the differences between successive prices [+1.50, −1.70, +2.40]. This removes the trend and makes the series stationary.

Diff W2 = 51.50 − 50.00 = +1.50

Diff W3 = 49.80 − 51.50 = −1.70

Diff W4 = 52.20 − 49.80 = +2.40

4️⃣ AUTO REGRESSIVE (AR) MODELS — Predict Using Your Own Past

WHAT: Regression on Yourself

Normal regression: Y depends on X₁, X₂, X₃ (different variables)

Auto regression: Y depends on its OWN past values (Y_{t-1}, Y_{t-2})

AR(1) FORMULA:

r_t = α + β × r_{t−1} + ε

Where: r_t = return this week, r_{t−1} = return last week, α = average return when no momentum, β = how much last week affects this week, ε = random noise

What β tells you:

β > 0 (e.g., 0.3)Trending/Momentum — positive follows positive
β < 0 (e.g., −0.2)Mean-reverting — positive followed by negative
β ≈ 0Random — yesterday tells us nothing about today

WORKED EXAMPLE (AR(1) using Core Data):

Suppose α = 0.001 (0.1%) and β = 0.25

Week 3 prediction:

r̂₃ = 0.001 + 0.25 × (+0.0300) = 0.001 + 0.0075 = +0.0085 = +0.85%

Actual r₃ = −3.30% → Error = −3.30% − 0.85% = −4.15%

Week 4 prediction:

r̂₄ = 0.001 + 0.25 × (−0.0330) = 0.001 + (−0.00825) = −0.00725 = −0.73%

Actual r₄ = +4.82% → Error = +4.82% − (−0.73%) = +5.55%

HOW: AR(1) Rolling Strategy in Code

from statsmodels.tsa.arima.model import ARIMA  # Import ARIMA

def create_ar_strategy(returns, window=100):
    signals = []                         # Store predicted signals
    for i in range(window, len(returns)):
        train = returns[i-window:i]      # Last 100 weeks
        model = ARIMA(train,
            order=(1, 0, 0))             # AR(1): p=1, d=0, q=0
        fitted = model.fit()             # Estimate α and β
        forecast = fitted.forecast(
            steps=1)[0]                  # Predict next week
        signal = 1 if forecast > 0 else -1  # Long or Short
        signals.append(signal)
    return signals
# Runs ~440 times (540 − 100), takes ~25 seconds

r_t = α + β₁ × r_{t−1} + β₂ × r_{t−2} + ε

Uses two past weeks. Set order=(2, 0, 0). Generalizes to AR(p) with p past values.

5️⃣ ACF & PACF — Choosing Your Model Order

WHAT: AutoCorrelation vs. Partial AutoCorrelation

ACF (AutoCorrelation Function): Measures correlation between r_t and r_{t−k} INCLUDING all indirect paths through intermediate lags. Like measuring how much Week 1 affects Week 5 — both directly AND through Weeks 2, 3, 4.

PACF (Partial AutoCorrelation Function): Measures ONLY the direct correlation between r_t and r_{t−k}, stripping out indirect effects. Like measuring how much Week 1 affects Week 5 directly — ignoring any influence through Weeks 2, 3, 4.

from statsmodels.graphics.tsaplots import plot_acf   # ACF plot
from statsmodels.graphics.tsaplots import plot_pacf  # PACF plot

fig, axes = plt.subplots(1, 2, figsize=(12, 4))     # Side by side
plot_acf(returns.dropna(), ax=axes[0], lags=15)      # ACF up to lag 15
plot_pacf(returns.dropna(), ax=axes[1], lags=15)     # PACF up to lag 15
plt.show()

WORKED EXAMPLE (auto-correlation by hand):

Returns: [+3.00, −3.30, +4.82, −2.30, +4.71, −2.43, +3.65]

Lag-1 pairs: (3.00,−3.30), (−3.30,+4.82), (+4.82,−2.30)…

If correlation is negative → mean-reverting pattern

HOW TO READ: Choosing p (AR) and q (MA)

🔑 THE MEMORY TRICK:

• ACF (starts with A) tells you the MA order (q) — NOT the AR order!

• PACF (starts with P) tells you the AR order (p)

"A goes with M, P goes with A" — the OPPOSITE of what you'd expect.

How to read the plots:

PlotWhat to Look ForDetermines
ACFLags that jut out of the blue shadowq (MA order)
PACFLags that jut out of the blue shadowp (AR order)

WORKED EXAMPLE (PayPal weekly):

Looking at ACF: Lag 1 nearly touches the blue band. Lag 2 is halfway. → q = 2

Looking at PACF: Same pattern — lag 1 significant, lag 2 halfway. → p = 2

For prices (non-stationary): d = 1 (one differencing operation)

→ ARIMA(2, 1, 2) is our starting point

💡 TIP — This is a heuristic, not gospel: ACF/PACF give a starting point. You should also try nearby orders (1,1,1), (2,1,1), (3,1,2) and compare performance. Yash's suggestion: run a nested for loop over all combinations!

💡 TIP — d in finance: For financial prices, d is almost always 1. No need to agonize over it. Returns = prices differenced once = d=1.

6️⃣ ARIMA MODELS — The Full AR + I + MA Framework

AR: AutoRegressive (Past Values)

WHAT: Uses past returns to predict current returns.

WHY: Captures momentum or mean-reversion behavior.

AR part of ARIMA(2,1,2):

β₁ × r_{t−1} + β₂ × r_{t−2}

WORKED EXAMPLE:

β₁=0.25, β₂=0.10

r_{t-1} = +3.00%, r_{t-2} = +3.00%

AR = 0.25×3.00 + 0.10×3.00

= 0.75 + 0.30 = +1.05%

I: Integrated (Differencing)

WHAT: Makes non-stationary data stationary by computing differences.

WHY: Prices trend; differences/returns don't. ARIMA handles this automatically when d=1.

d=0: process is already stationary

d=1: difference once (prices → returns)

d=2: difference twice (rare in finance)

WORKED EXAMPLE:

Prices: 50.00, 51.50, 49.80

Diff(d=1): +1.50, −1.70

Diff(d=2): −1.70−(+1.50) = −3.20

💡 KEY: With d=1, you can feed prices directly to ARIMA — it differences internally. No need to compute returns yourself.

MA: Moving Average (Past Errors)

WHAT: Uses past prediction errors to improve current prediction. NOT the SMA/LMA moving average from trading!

WHY: If a model systematically over/under-predicts, MA corrects for that pattern.

MA part of ARIMA(2,1,2):

γ₁ × ε_{t−1} + γ₂ × ε_{t−2}

WORKED EXAMPLE:

γ₁=0.30, γ₂=0.15

ε_{t-1} = −4.15%, ε_{t-2} = +2.00%

MA = 0.30×(−4.15) + 0.15×(+2.00)

= −1.245 + 0.30 = −0.945%

⚠️ WARNING: The "MA" in ARIMA has NOTHING to do with SMA/LMA moving average crossovers. ARIMA's MA = moving average of errors.

FULL ARIMA(2,1,2) EQUATION:

P_t = α + β₁ × P_{t−1} + β₂ × P_{t−2} + ε_t + γ₁ × ε_{t−1} + γ₂ × ε_{t−2}

"Fitting" the model = estimating α, β₁, β₂, γ₁, γ₂ from historical data. Python does this for you.

7️⃣ ROLLING ARIMA STRATEGY — Implementation & Performance

HOW: The Rolling Forecast Code

WHAT: At each week, fit ARIMA on the last 100 weeks and predict ONE week ahead. If predicted price > current price → go long. Else → go short.

from statsmodels.tsa.arima.model import ARIMA  # ARIMA model

def create_arima_strategy(prices, window=100):
    signals = []                         # Store signals
    predictions = []                     # Store forecasts

    for i in range(window, len(prices)):
        train = prices[i-window:i]       # Last 100 weeks
        model = ARIMA(train,
            order=(2, 1, 2))             # ARIMA(2,1,2)
        fitted = model.fit()             # Estimate coefficients
        forecast = fitted.forecast(
            steps=1)[0]                  # 1-step forecast
        last_price = train.iloc[-1]      # Current week's price

        # Signal: long if forecast > current price
        signal = 1 if forecast > last_price else -1
        signals.append(signal)
        predictions.append(forecast)

    return signals, predictions
# Takes ~90 seconds for ~440 iterations

WORKED EXAMPLE — One Forecast Step:

Train data: Weeks 1–100 (prices)

ARIMA(2,1,2) fitted → coefficients estimated

Last price (Week 100): $51.72

Forecast (Week 101): $52.78

$52.78 > $51.72 → Signal = +1 (LONG)

Next iteration: train = Weeks 2–101, forecast Week 102…

Performance Evaluation

# Compute strategy returns
data['strategy_returns'] = (
    data['signal'].shift(1) *     # Previous signal
    data['returns']               # Current returns
)

# Total return (simple compounding)
total = (1 + data['strategy_returns']).cumprod().iloc[-1] - 1

# Sharpe ratio (annualized for weekly)
sharpe = (data['strategy_returns'].mean() /
    data['strategy_returns'].std()) * np.sqrt(52)

# Win rate
wins = (data['strategy_returns'] > 0).sum()
total_trades = data['strategy_returns'].notna().sum()
win_rate = wins / total_trades * 100

# Maximum drawdown
cum_returns = (1 + data['strategy_returns']).cumprod()
peak = cum_returns.cummax()
drawdown = (cum_returns - peak) / peak
max_dd = drawdown.min() * 100

WORKED EXAMPLE — Sharpe Ratio:

Mean weekly return = 0.076%

Std weekly return = 2.03%

Weekly Sharpe = 0.076 / 2.03 = 0.037

Annualized = 0.037 × √52 = 0.037 × 7.211 = 0.27

This now matches the lecture's reported Sharpe ratio of about 0.27 after annualizing weekly returns.

💡 WHY ROLLING FORECASTS MATTER: A static model trained once on 2015–2020 data will fail on 2021+ data. Rolling forecasts retrain the model weekly, so it adapts to new market conditions — the key advantage over fixed rules.

🎯 KEY TAKEAWAYS

1. Simple MA rules fail during regime shifts (COVID, wars, rate hikes). Statistical models like ARIMA adapt because they retrain on recent data.

2. Always check stationarity first (ADF test: p < 0.05). Model returns (stationary), not prices (non-stationary) — or use d=1 to let ARIMA difference automatically.

3. ACF tells you MA order (q), PACF tells you AR order (p). Memory trick: "A→M, P→A" — the opposite letters. In finance, d is almost always 1.

4. Use rolling one-step-ahead forecasts, never static models. Even ARIMA has severe drawdowns — it's a starting point, not a production-ready strategy.

THE MASTER FORMULA: ARIMA(p, d, q): P_t = α + [β₁·P_{t-1} + β₂·P_{t-2}] + [ε_t + γ₁·ε_{t-1} + γ₂·ε_{t-2}] — AR captures momentum, I handles non-stationarity, MA corrects prediction errors

THE ONE THING TO NEVER FORGET: ARIMA's "Moving Average" is about error correction, NOT trading moving averages (SMA/LMA). And always use ROLLING forecasts — a model trained once is a model that will fail.
