1 · Core Example — K-Means Clustering for Stock Portfolio Grouping

WHAT

K-Means clustering groups 500 Nifty stocks into clusters using quantitative features (returns & volatility) instead of traditional sector/industry labels. Each cluster reveals stocks with similar risk-return profiles, enabling differentiated strategy application or position sizing.

WHY

Traditional grouping by sector or market cap is the common starting point. Quantitative clustering can uncover similarities that those labels miss. It enables: (a) different strategies per cluster, (b) adaptive position sizing, (c) statistical arbitrage pair discovery, (d) mean-variance portfolio construction.

HOW — Full Pipeline

# Step 1: Download 1 year of data for 500 stocks
import yfinance as yf                         # Yahoo Finance library
tickers = [...]                                # 500 Nifty constituents
data = yf.download(tickers, period="1y")["Adj Close"]  # adjusted close prices
data = data.dropna(axis=1)                     # axis=1 → drop COLUMNS with ANY missing data (416 remain)

# Step 2: Build features — annual returns & annualized volatility
returns = (data.pct_change() + 1).prod() - 1   # total net return over 1 year
volatility = data.pct_change().std() * (252**0.5)  # daily std × √252 → annualized vol

# Step 3: Scale features (MANDATORY for distance-based algorithms)
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
features_scaled = scaler.fit_transform(           # mean=0, std=1 for both features
    pd.DataFrame({"returns": returns, "volatility": volatility})
)

# Step 4: Fit K-Means with 3 clusters
from sklearn.cluster import KMeans
model = KMeans(n_clusters=3, random_state=1)   # 3 clusters, reproducible
model.fit(features_scaled)                     # train — no train/test split needed
labels = model.labels_                         # array: [0, 1, 2, 0, 2, ...] per stock

# Step 5: Assign clusters back to DataFrame & visualize

plt.scatter(df["volatility"], df["returns"], c=df["cluster"], cmap="viridis")
# Output: 3 colored clusters — e.g., low-vol/positive-return vs high-vol/negative-return
Cluster 0: Low vol + positive returns → could justify increasing position size
Cluster 1: Medium vol + flat returns → could justify keeping position size similar
Cluster 2: High vol + negative returns → could justify reducing position size
GOTCHA: Without scaling, debt-to-equity (mean=103) dominated clustering over beta (mean≈1) and created nearly horizontal clusters. In this distance-based setup, scaling is important before K-Means.

2 · K-Means Clustering Algorithm — How It Works

WHAT

K-Means is an unsupervised algorithm that partitions N observations into K clusters by minimizing total squared distances between points and cluster centroids. It iterates: assign → recalculate centroid → reassign until centroids stabilize.

STEP-BY-STEP

  1. Choose K (number of clusters)
  2. Initialize K random centroids
  3. Calculate distance from each point to all K centroids
  4. Assign each point to nearest centroid
  5. Recalculate centroid = mean of assigned points
  6. Repeat steps 3-5 until centroids don't move

In the lecture's examples, convergence was usually fast, often within a handful of iterations.

WHY — Clustering Applications in Trading

  • Strategy allocation: apply different strategies to different clusters
  • Position sizing: reduce size for high-vol clusters, increase for low-vol
  • Stat arb pairs: create N/2 clusters → each cluster = 2 stocks → check cointegration & z-score
  • Portfolio construction: treat each cluster as a portfolio → run mean-variance optimization across clusters

Cluster bounds: min = 1, max = N (number of observations). Useful range found via Elbow Method.

WORKED EXAMPLE — Elbow Method

# Test K=1 to K=20, record inertia
inertia_list = []              # stores sum of squared distances
for k in range(1, 21):        # 1 cluster to 20 clusters
    km = KMeans(n_clusters=k, random_state=1)
    km.fit(features_scaled)    # train model
    inertia_list.append(km.inertia_)
    # inertia_ = Σ(dist² from each point
    #   to its assigned centroid)

# Plot: x=K, y=inertia → look for "elbow"
plt.plot(range(1,21), inertia_list, marker="o")
plt.xlabel("Number of Clusters")
plt.ylabel("Model Inertia")
# Elbow at K=3 or K=4 → candidate range to inspect
# Too few → limited insight
# Too many → less meaningful groupings

TERMINOLOGY: Model inertia = sum of squared distances from each observation to its centroid. Centroid = mean position of cluster members.

GOTCHA: No automated "right K" — use Elbow Method + judgment. Adding 1 more cluster that contains only 1 data point is meaningless. Some practitioners now feed elbow charts to Large Language Models (LLMs) for K recommendation.

3 · From Biological to Artificial Neurons — The Building Block

WHAT — Biological Neuron

Three parts: Dendrite (receives info from senses) → Nucleus (processes info, decides importance) → Axon Terminal (passes processed info to next neuron). Average adult brain: 86 billion neurons taking collective decisions — no single neuron decides alone.

Different sensory inputs (touch, sight, hearing, smell, taste) activate different neuron networks in different brain regions. Each neuron processes a part of the information.

HOW — Artificial Neuron Replication

Map biological → artificial:

BiologicalArtificialFunction
DendriteInputs x₁..xₙReceive features
NucleusZ = Σ(xᵢ × wᵢ)Weighted sum
AxonY = f(Z)Activation → output

Linear function: Z = x₁w₁ + x₂w₂ + ... + xₙwₙ + bias

Weights (w) = importance of each feature. We do NOT set weights — the neural network learns them. If we knew the right weights, we wouldn't need ML.

WORKED EXAMPLE

# Artificial neuron with 3 features
# Features: x1=0.5, x2=-0.3, x3=0.8
# Weights (learned): w1=0.7, w2=-0.4, w3=0.6
# Bias: b = 0.1

Z = (0.5*0.7) + (-0.3*-0.4) + (0.8*0.6) + 0.1
# Z = 0.35 + 0.12 + 0.48 + 0.1 = 1.05

# Apply sigmoid activation:
import math
Y = 1 / (1 + math.exp(-1.05))  # sigmoid(1.05)
# Y = 0.741 → >0.5 → predict class 1 (UP)

# Apply tanh activation:
Y_tanh = math.tanh(1.05)
# Y_tanh = 0.783 → >0 → predict +1 (LONG)

# Apply ReLU activation:
Y_relu = max(0, 1.05)
# Y_relu = 1.05 → numerical prediction

TERMINOLOGY: Weight = importance assigned to a feature. Bias = intercept term (like regression error term). Activation function = non-linear transform of Z that determines if neuron "fires."

4 · The Four Activation Functions That Matter

SIGMOID

Formula: f(z) = 1 / (1 + e⁻ᶻ)

Output range: [0, 1]

Use: Output layer for buy/hold classification (0 = hold, 1 = buy). Also called logistic function.

Behavior: z=5 → 0.993, z=0 → 0.5, z=−5 → 0.007. Squashes any input to 0-1 probability.

TANH (Hyperbolic Tangent)

Formula: f(z) = (eᶻ − e⁻ᶻ) / (eᶻ + e⁻ᶻ)

Output range: [−1, +1]

Use: Output layer for long/short classification (+1 = long, −1 = short).

Behavior: z=5 → 1.0, z=0 → 0.0, z=−5 → −1.0. Centered at zero unlike sigmoid.

ReLU (Rectified Linear Unit)

Formula: f(z) = max(0, z)

Output range: [0, ∞)

Use: A common choice for hidden layers in practice. Also usable in the output layer for regression tasks such as price or volatility prediction.

Behavior: z=5 → 5, z=0 → 0, z=−5 → 0. Passes positive values unchanged, zeros out negatives.

LEAKY ReLU

Formula: f(z) = max(αz, z), α ≈ 0.01

Output range: (−∞, ∞)

Use: Hidden layers when ReLU causes "dead neurons" (all-zero outputs). Small α prevents mathematical issues with pure zeros.

Behavior: z=5 → 5, z=0 → 0, z=−5 → −0.05. Like ReLU but leaks a tiny value for negatives.

DECISION RULE — Which Activation Where?

LayerTaskActivationReason
HiddenAnyReLU / Leaky ReLUCommon empirical default in the lecture
OutputBuy/Hold (0,1)SigmoidBounds to [0,1]
OutputLong/Short (+1,−1)TanhBounds to [−1,+1]
OutputRegression (price)ReLU / LinearNo bounding needed

KEY RULES

  • Activation is set at the LAYER level — all neurons in a layer share the same activation
  • CANNOT assign different activations to individual neurons within a layer
  • Different layers CAN have different activation functions
  • Hidden: ReLU. Output: depends on task. This covers 90%+ of use cases
  • No activation = linear function = input equals output (used for regression output)

GOTCHA: Exponential activation functions would "explode" the algorithm — weights can't converge. That's why ReLU (simple max) dominates hidden layers.

5 · Network Architectures — From Perceptron to Deep Networks

WHAT — Three Architecture Levels

ModelStructureCapability
PerceptronInput → 1 neuron → OutputToo simple, learns nothing useful
Feed ForwardInput → 1 hidden layer → OutputBasic pattern recognition
Deep Neural NetworkInput → N hidden layers → OutputComplex non-linear patterns

"Deep" = information passes through multiple layers before producing output. More layers = more abstract feature combinations learned.

WHY — Multiple Neurons with Same Input?

All neurons in a layer receive the SAME features, but each neuron has DIFFERENT weights. So each neuron perceives the data differently.

Same input + different weights = different Z → different activation → different output per neuron

This mimics biological brains: same sensory input goes to many neurons, each processes a different aspect. Collective decision is more powerful than any single neuron.

Layer sizing: No formula for optimal neurons per layer or number of hidden layers. Determined by trial and error. Varies by problem, data size, and feature count.

WORKED EXAMPLE — Layer Specification

# Perceptron: 1 neuron (too simple)
MLPClassifier(hidden_layer_sizes=(1,))

# Feed Forward: 1 hidden layer, 5 neurons
MLPClassifier(hidden_layer_sizes=(5,))
# Architecture: Input(12) → Hidden(5,ReLU) → Output(1)

# Deep Network: 3 hidden layers
MLPClassifier(hidden_layer_sizes=(5,4,3))
# Architecture:
#   Input(12 features)
#   → Hidden1(5 neurons, ReLU)
#   → Hidden2(4 neurons, ReLU)
#   → Hidden3(3 neurons, ReLU)
#   → Output(1 neuron, tanh)

# Weights per connection:
# Input→Hidden1: 12×5 = 60 weights
# Hidden1→Hidden2: 5×4 = 20 weights
# Hidden2→Hidden3: 4×3 = 12 weights
# Hidden3→Output: 3×1 = 3 weights
# Total: 95 weights + biases to learn

6 · Training Process — Forward & Backward Propagation

WHAT — The Training Loop

  1. Initialize all weights randomly
  2. Forward Propagation: data flows Input → Hidden → Output → predictions made
  3. Loss Function: compare predictions vs actual target values. Higher loss = bigger error
  4. Backward Propagation: loss feeds back to optimizer
  5. Optimizer (e.g., Adam / Gradient Descent) produces NEW weights for ALL neurons
  6. Repeat steps 2-5 with updated weights
  7. Stop when: (a) loss can't improve for N iterations, or (b) max iterations reached

Each complete pass = 1 iteration/epoch. The optimizer is the "core" of neural networks — it's what makes learning possible.

WHY — How the Optimizer Learns

The optimizer (Gradient Descent / Adam) uses the loss score as feedback:

  • If loss increased after weight update → do the OPPOSITE direction
  • If loss decreased → continue in that direction & calibrate further
  • Process converges when no weight adjustment can reduce loss further

Key insight: The network never finds weights in one pass. It iteratively refines through hundreds of forward-backward cycles. Example: started at 82% loss → decreased to 69% by iteration 20 → converged at iteration 121 (stopped because last 10 iterations showed no improvement).

TERMINOLOGY: Gradient Descent = mathematical method to find minimum of loss function by following the slope. Adam = Adaptive Moment Estimation, an enhanced version of gradient descent. Epoch = one complete pass through all training data.

WORKED EXAMPLE — Training Output

# Define & train a neural network
from sklearn.neural_network import MLPClassifier
model = MLPClassifier(
    hidden_layer_sizes=(5,),  # 1 layer, 5 neurons
    max_iter=300,             # max 300 weight updates
    activation='relu',        # hidden layer activation
    solver='adam',            # optimizer algorithm
    shuffle=False,            # don't shuffle rows
    verbose=True              # print training progress
)
model.fit(X_train_scaled, y_train)
# Iteration 1, loss = 0.8237     ← high loss (random weights)
# Iteration 20, loss = 0.6913    ← improving
# Iteration 121, loss = 0.6801   ← converged
# Training loss did not improve for 10 iterations
# → model.score(X_train, y_train) = 0.57  (57%)
# → model.score(X_test, y_test)  = 0.53  (53%)

GOTCHA: We set max_iter=300 but model stopped at 121 — it auto-stops when loss plateaus for 10 consecutive iterations. This prevents wasted computation.

8 · Portfolio-Level ML & Practical Deployment Insights

WHAT — Why Portfolio > Individual

  • The lecture suggests neural networks usually need very large datasets, often hundreds of thousands of rows
  • Single-instrument daily data, even across decades, is usually too small for this style of modeling
  • Combining intraday data across many instruments is one practical way to reach usable data volume
  • Portfolio-level prediction can benefit from diversification, which may help contain drawdowns
  • Single-instrument models tend to show larger drawdowns and less reliable signals in the lecturer's experience

Data sources: US daily data available from 1873. Decent quality from 1950s. But daily resolution is still insufficient for deep learning.

WHY — Industry Trend Shift

Machine Learning (ML) in trading is shifting from:

Old FocusNew Focus
Predict price directionPredict volatility of volatility
Individual instrumentPortfolio-level signals
Generate buy/sell signalsPredict position SIZE
Binary directionsDistribution of order flow

The lecturer's view is that practitioners are increasingly predicting derivatives of derivatives — signal distributions, bet sizes, and risk metrics — rather than just raw price direction.

KEY TAKEAWAYS

Clustering Power

K-Means discovers hidden stock groups using quantitative features. Scale features first. Use the Elbow Method plus judgment when choosing K. Applications: strategy allocation, position sizing, stat arb pair discovery.

Neural Network Essence

Artificial neurons replicate biology: inputs × weights → linear function Z → activation function → output. Weights are LEARNED (not set by us). Same input + different weights = different perception per neuron.

Activation = Task Selector

Hidden layers in this lecture usually use ReLU or Leaky ReLU. Output layer: Sigmoid (buy/hold 0,1), Tanh (long/short −1,+1), ReLU or Linear (regression). Activation is set per layer, not per neuron. These were the four functions emphasized in practice.

Portfolio > Individual

Deep learning typically benefits from very large datasets. Single-stock daily data is usually insufficient, so combining instruments at portfolio level is more practical. The lecturer also notes an industry shift from predicting direction toward predicting position size, risk, and signal distributions.
