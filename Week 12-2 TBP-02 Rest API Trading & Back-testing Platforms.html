THE PROBLEM
The IB TWS API (TBP-01) only talks to IB. You want to automate across many brokers with one skill.
YOUR GOAL
Master REST: 5 request pieces, 2 response parts, JSON, and a heartbeat to stay connected.
THE EDGE
REST is the de-facto standard — learn once, connect to virtually any broker (post-COVID).

⭐ CORE PATTERN — every request needs 5 pieces; every response has 2 parts

REST = REpresentational State Transfer: the client asks for the state of a variable (PnL, positions, price); the server reads it and transfers a representation back. It is synchronous: request → wait → response → act. The whole URL = base URL + endpoint.

① API vs REST

TermMeaning
APIprogrammable client↔server bridge
RESTdesign principles (stateless, uniform, cacheable, client-server) → comes as HTTP URLs
TBP-01 IB API = a Python library; REST = HTTP URLs. As end users we don't care about the principles — just base URL + endpoint.

② The 5 request pieces

#PieceWhat
1Base URLbroker server address (docs p.1)
2Endpoint"department" per op: /pnl
3HTTP methodGET=fetch / POST=send
4Headersabout the client (auth/type), a dict
5Payload/bodyabout the request (symbol,qty), JSON

③ The response — 2 parts

1st digitMeaningFault
2xxsuccess—
4xxbad URL/method/payload (404)you
5xxserver can't copebroker
💡 Check the status code FIRST, then read the content. Codes are an HTTP-consortium standard — universal across the whole web.

④ The code pattern

import requests, json
BASE = "https://localhost:5000/v1/api"
url  = BASE + "/pnl"                 # 1+2
headers = {"Content-Type":"application/json"}   # 4
payload = {"symbol":"NIFTY"}        # 5
r = requests.get(url, headers=headers,
                 json=payload, verify=False)  # 3
print(r.status_code)                # part 1
data = json.loads(r.content)        # part 2
⚠️ verify=False is IB-specific — its local gateway has no SSL cert. Other brokers use a hosted URL with SSL.

⑤ JSON — the common language

Your dicts are Python-only; the server may be C++. JSON (JavaScript Object Notation — not JavaScript) is a neutral text format everyone understands.

methoddirection
json.dumps()dict → JSON (send)
json.loads()JSON → dict (receive)

⑥ What REST is NOT

✗ Not for HFT — internet latency (seconds). HFT = FIX + FPGA. REST is for LFT/MFT (~1s).
✗ Not uniform — each broker exposes different ops; read the docs.
✗ Not a replacement — complements the web/desktop terminal you keep for monitoring.

⑦ Staying connected

🔑 IB drops idle sessions. Fix: call /reauthenticate, or run a continuous tickle (heartbeat ~every 55s) on a separate thread. You may NOT auto-login (regulatory) — log in manually.

Continuation addendum - REST vs WebSocket

Practical split: use REST for auth checks, historical bars, positions, PnL, and orders. Use WebSockets when you need a continuing stream of last price, bid, ask, PnL, or order updates after one subscription.
