Basics of Python Programming

Week 5-1 · PBQ-01 — your first coding lecture, written as a lesson you can read end to end.

Up to now this course has lived in spreadsheets and theory. This lecture is the turning point: you start writing code. The goal is not to memorise syntax but to become fluent enough to read a strategy and know whether its logic is correct. Even in an age where AI writes code for you, that judgement is the skill that matters — and it only comes from typing code yourself. The lecturer's advice is blunt and worth repeating: practise about an hour a day, one topic at a time, until it becomes muscle memory.

We'll walk the same path as the live session: printing, then variables, then the three core data structures, then NumPy and pandas, and finally the thing all of it is for — turning conditions into trading signals on real TCS stock data.

1. Why Python, and how it runs

Python is a general-purpose, open-source language prized for readability and for its enormous quant ecosystem — pandas, numpy, matplotlib, scikit-learn and more. It is an interpreted language: an interpreter reads your code and translates it to machine instructions line by line, which is why you get immediate feedback in a notebook.

You can run Python several ways: an interactive shell, a Jupyter notebook (what we use), Google Colab, plain .py scripts, or a full IDE like VS Code. In Jupyter, a notebook is a stack of cells. Code cells run code; markdown cells hold explanations. There are two modes: edit mode (click into a cell and type) and command mode (press Esc, then navigate). A few shortcuts pay for themselves immediately:

2. Printing — making output tell a story

A raw number on screen is nearly useless in live trading; you want output that reads like "Bought TCS at 3,512 at 09:21." The print function gives you that control. The lecture shows three styles using Intel's market capitalisation.

firm_name = "Intel Corporation"
market_cap = 105.95 * 10**9
shares = market_cap / 25.47        # ≈ 4,159.8 million

print("Mkt cap of", firm_name, "is US$", market_cap)          # comma style
print(f"ticker {ticker}  price US$ {last_price}")             # f-string
print("{0:s} has {1:.1f} million shares".format(firm_name, shares/10**6))

The third line is the most precise. The f-string (a string prefixed with f) lets you drop variables straight into { }. The .format method uses numbered placeholders with optional type and precision codes. Here is exactly how that mapping works:

PlaceholderTakes which argumentHow it renders
{0:s}argument 0 — firm_nameas a string → "Intel Corporation"
{1:.1f}argument 1 — 4159.7958float, 1 decimal → 4159.8
{1:.2f}same argumentfloat, 2 decimals → 4159.80
Remember: indexing starts at 0. :s means string, :.1f a float to one decimal, :d an integer. Anything outside the braces is printed literally.

3. Variables and dynamic typing

You never declare a type in Python. Assign a value and Python infers it; reassign and the type can change. The = sign is assignment, not equality.

x = 5        # int
x = 5.5      # float
x = "five"   # str

You can list everything in memory with the magic commands %who (names) and %whos (names with types and values) — handy when a notebook grows long.

4. The three native data structures

Python ships with three collections you will use constantly. The difference between them is small but important.

StructureBracketsChangeable?Typical use
list[ ]yes (mutable)an ordered, editable collection
tuple( )no (immutable)a fixed set that must not change
dictionary{ }yeslook up a value by a unique key

All three can hold mixed types. Lists grow with .append (add at the end) and .insert (add at a position); len() counts them. A tuple looks like a list but is frozen — try .append on one and Python tells you the method doesn't exist. A dictionary stores key: value pairs, which is perfect for trading mappings like symbol → price or order-id → fill details.

prices = {"TSLA": 248.5, "INTC": 25.47, "AAPL": 189.9}
prices["INTC"]          # → 25.47   (look up by key)
prices.keys()           # all symbols
prices.values()         # all prices
The tuple trap: once you write t = ("NSE","NYSE") you cannot append, delete, or edit it. Use a tuple precisely when you want that protection.

Indexing and slicing

Positions count from 0 on the left and from -1 on the right. A slice a:b includes a but excludes b — the single most common beginner surprise.

bp = ["Oil, Gas and Coal", "Energy", 290.0, "LSE", "GBp"]
bp[0]      # 'Oil, Gas and Coal'  (first)
bp[-1]     # 'GBp'                (last)
bp[1:3]    # ['Energy', 290.0]    (1 and 2; 3 excluded)
"LSE" in bp   # True              (membership test)

5. NumPy — the reason arrays beat lists

NumPy ("numerical Python") adds the array, an ordered collection whose elements are all the same type. That homogeneity unlocks vectorised maths: an operation applies to every element at once. Compare:

[1.65, 1.80] * 2                 # → [1.65, 1.80, 1.65, 1.80]  (list duplicates!)
np.array([1.65, 1.80]) * 2       # → [3.30, 3.60]              (array scales)
This is not a curiosity — it is why every back-test uses arrays. To multiply each return by a weight, or each price by a factor, an array does it in one expression; a list cannot.

NumPy also simulates data. np.random.normal(loc, scale, size) draws from a normal distribution — loc is the mean, scale the standard deviation, size the count. One line gives you 10,000 points; a spreadsheet would need 10,000 drags. To find things, np.where(condition) returns the indices where the condition holds, and you can read the values back with vol[idx].

Note that mathematical helpers like np.mean and np.median belong to NumPy, not to plain lists — that is why you call them as np.mean(arr) rather than arr.mean() on a list.

6. pandas — loading real market data

Where NumPy handles arrays of numbers, pandas handles labelled tables. Its DataFrame is a spreadsheet in code: rows and named columns, with an index down the side. We load the lecture's TCS file and make the date the index.

import pandas as pd
df = pd.read_csv("TCS.NS.csv", index_col="Date", parse_dates=["Date"])
df.shape       # (2460, 6)   — 2,460 days, 6 columns

7. Loops and conditionals

A for loop repeats an action over a sequence — the code equivalent of dragging a formula down 10,000 spreadsheet rows. range(n) produces 0, 1, … n-1 (the end is excluded, just like slicing).

for i in range(10):
    print(i * 2)            # 0, 2, 4, … 18

When you need both the value and its position — for instance, which day a price crossed a level — wrap the sequence in enumerate, which hands you the index and the value together. Pair it with if / elif / else to make decisions:

for day, price in enumerate(closes):
    if   price > 83: signal = "SELL"
    elif price < 67: signal = "BUY"
    else:            signal = 0
np.where is simply a one-line version of this if/else: "where the condition is true put A, otherwise put B." Both express the same logic; np.where just does it across a whole array at once.

8. The payoff — Python as a signal generator

Everything above exists to support this moment. A trading signal is just a condition encoded as a number — typically +1 for long, -1 for short, 0 for flat. The lecture builds them two ways. First, vectorised on simulated volume:

signals = np.where(vol > 325, "short", "0")     # high volume → short
signals = np.where(vol < 175, "long",  signals) # low volume  → long
np.unique(signals, return_counts=True)          # count each signal

Then, the real thing — a moving-average crossover on our 2,460 TCS bars. We go long whenever the fast 20-day average sits above the slow 50-day average:

d["SMA20"] = d["Close"].rolling(20).mean()
d["SMA50"] = d["Close"].rolling(50).mean()
d["signal"] = np.where(d.SMA20 > d.SMA50, 1, 0)   # 1 = long, 0 = flat

Across the full TCS history this rule produced 34 long-entry crossovers and kept us in the market 1,456 of 2,460 days (about 59%). The shaded regions in the practice notebook's chart show exactly when the fast line led the slow one. That is the whole arc of the lecture: a few lines of Python, applied to real prices, turn raw data into actionable decisions.

How to practise: open the practice notebook, change a number, and rerun. Make the crossover 10/30 instead of 20/50. Swap TCS for another CSV. The fastest way to learn is to break something and fix it — about an hour a day, one topic at a time.

Key takeaways

  • Output: print, f-strings (f"{var}"), and .format with {index:type.precision} for clean, labelled results.
  • Typing is dynamic: = assigns; the value decides the type.
  • Three structures: list [] (mutable), tuple () (immutable), dict {} (key→value). Indexing from 0; slices exclude the end.
  • NumPy arrays vectorise maths (a list *2 duplicates; an array *2 scales); np.where finds conditions, np.random.normal simulates.
  • pandas read_csv loads market data; shape, head, describe inspect it.
  • Loops & conditionals automate repetition; a signal is a condition turned into +1/0/−1 — the seed of every strategy to come.
