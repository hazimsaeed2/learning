Natural Language Processing & Sentiment Analysis in Trading

As alpha from market-discovered data (price, volume, the order book — each carrying a ground truth) decays, funds turn to alternative data: news, sentiment, transcripts, ESG. It is modelled, fuzzy and subjective — five ESG vendors give a stock five scores. This lecture is the assembly line that turns words into numbers into signals: tag → cleanse → embed → score → smooth → test → trade.

2 · From a news item to a score  (step calculation)

A vendor tags each item with the companies it mentions (each gets a permID), then a model emits three probabilities; the argmax becomes the class. One item → many companies (one-to-many).

CompanyP(neg)P(neu)P(pos)argmaxscore
Innovent Biologics0.620.300.08neg−1
WuXi Bio0.180.550.27neu0
Royal Dutch Shell0.100.270.63pos+1
BP0.410.340.25neg−1

Without a common identifier the news→sentiment mapping is impossible. There is no industry-wide ID (Bloomberg / FactSet / S&P each have their own) — map via ISIN. LSEG's open permID is a 10-digit "barcode" for organisations, quotes and instruments.

4 · Text → numbers, and why bag-of-words misses meaning

"the company went bankrupt…" and "the company got delisted" mean the same thing but share no content words. TF-IDF sees only the function words "the company"; a semantic embedding sees the meaning. (TF-IDF value computed live in the notebook.)

PairTF-IDF cosineSemantic embeddingReading
bankrupt vs delisted0.34≈ 0.83only "the company" shared
bankrupt vs record profits0.00lowopposite meaning

Embedding dimensions: OpenAI 1536 · sentence-transformers 384 · BERT 768. Cosine similarity on those vectors captures semantics that a word-count vector throws away — that gap is why embeddings exist.

5 · Build your own sentiment model — out-of-box vs fine-tuned  (confusion-matrix diagonals)

Out-of-the-box LLMs are neutral-biased on financial text — they dump uncertain cases into "neutral". Fine-tuning (tweak only the final layers — cheap) on labelled data lifts the positive/negative hit-rate sharply.

True classOut-of-box diagonalFine-tuned diagonalLift
positive29%59%+30 pp
negative31%74%+43 pp
neutralhighhighalready easy

These are the instructor's own figures on his hand-labelled financial set — a proprietary corpus that can't be reproduced offline. The tedious, irreplaceable step is labelling — human experts; you cannot use an LLM to improve an LLM. Split train / validation / test as always.

6 · Sentiment is noisy → state-space (Kalman) smoothing  (real Baidu feed)

Raw daily sentiment is volatile. A local-level model (statsmodels UnobservedComponents) strips the noise before you overlay it on price. Numbers below are computed on the real Baidu sentiment history shipped with the lecture (4 459 scores → 1 674 trading days).

QuantityValueMeaning
raw daily net-sentiment std0.417what the feed prints
Kalman-smoothed std0.286latent mood recovered
corr(smoothed, price)+0.20a mild positive overlay

Baidu's overlay is only mildly positive — exactly the lecture's point that it is asset-specific: Alibaba's sentiment tracks price closely, China Mobile's barely moves, Facebook's is negatively correlated. Read every overlay one name at a time.

7 · Does sentiment lead price? — Granger causality  (real Baidu)

The million-dollar question: if price already drives sentiment, the signal is already priced in and useless. Granger tests whether lags of one series help predict the other — here on Baidu daily net sentiment vs returns.

Directionbest p-value (lags 1–3)Reading
sentiment → returns0.046only marginally significant
returns → sentiment0.010reverse channel a touch stronger

For Baidu the two directions are comparable (news lags price about as much as it leads) — an honest no clean edge result. You want a strongly one-sided low p-value; symmetric p-values like these mean no exploitable lead-lag on this name.

10 · Topic modeling — file each item under a code

60–70% of NLP-in-finance work is on nested topic codes (markets → FX → FX-derivatives). History: LSA 1988 (matrix factorisation) → PLSA 2000 → LDA 2003 (Blei, took off ~2010). One scikit-learn call today.

LDA topic (unsupervised)Top words recovered
topic 0cloud, ai, chip, revenue, semiconductor, maker
topic 1demand, energy, oil, natural, rose, gas
topic 2loan, bank, supply, credit, defaults, refinery

CountVectorizer → document-term matrix → LatentDirichletAllocation. Topic codes are vendor-specific (not industry standard) and let you filter news by topic / sector / geography. Knowledge graphs (RDF triples; Neo4j / Neptune / TigerGraph) capture supply-chain propagation for low/mid-frequency data.

9 takeaways

  1. Alt-data is modelled, fuzzy, subjective — no ground truth; it complements decaying price-alpha.
  2. The sentiment record: news → permID-tagged companies → P(neg/neu/pos) → −1/0/+1 (one-to-many).
  3. Cleanse first: lowercase, strip punctuation/digits, remove stop-words, lemmatize.
  4. Bag-of-words cosine misses meaning (0.34 — only function words shared); semantic embeddings capture it (≈0.83).
  5. Out-of-box LLMs are neutral-biased (~29/31%); fine-tuning lifts positive/negative hit-rate to 59/74% (lecture).
  6. Raw sentiment is noisy → smooth with a state-space / Kalman filter (real Baidu overlay corr +0.20).
  7. Granger-test the lead-lag — if price drives sentiment, the signal is already priced in.
  8. Trade relative ranks, filter by buzz — absolute sentiment values are meaningless.
  9. Topic modeling (LSA→PLSA→LDA) files items under codes; labels are the irreplaceable ground truth.
