THE THREE DOF IN BUY-LOW-SELL-HIGH

# The model's decision line:
if x < -0.011:     # x = close_today - close_yesterday (%)
    go_long()       # DOF 1: close_today
                    # DOF 2: close_yesterday (lag-1, independent)
else:               # DOF 3: rate = 0.011 (tuned by sweeping
    go_short()      #         -0.02 to +0.02 → parameter tuning
                    #         = 1 extra DOF consumed)

💡 WHY RATE IS A DOF: Even though 0.011 is a fixed number in the final model, it was FOUND by sweeping multiple values and picking the best → used extra information → consumed 1 degree of freedom.

RULE OF THUMB: Data Points per DOF

Use CaseRecommended Data/DOF
Typical linear regression20–100
Stock price prediction≥ 1,000
ChatGPT (text generation)~5 (but different use case)

Why 1,000 for stocks? Price is non-normal, highly fluctuating. 100 is too low — you need more data to overcome the noise.

DECISION FRAMEWORK:

Data/DOF < 20 → OVERFITTING (memorized data)

Data/DOF = 20–100 → Depends on use case

Data/DOF = 100–1000 → Right-fitting (general)

Data/DOF > 1000 → Right/underfitting for stocks

⚠️ OVERFITTING ≠ "bad backtest vs good live". That's a SYMPTOM, not a definition. The MATHEMATICAL definition: too many parameters relative to training data → model memorizes instead of learning patterns.

3️⃣ LIVE TRADING ROBOT — Implementation, Assumptions & Error Handling

CORE ROBOT (2 Functions)

def initialize(context):
  # Runs ONCE at startup
  context.security = symbol('SPY')
  # Run buy_low_sell_high every day
  # at market_close - 1 minute (15:59 ET)
  schedule_function(
      buy_low_sell_high,
      date_rule=date_rules.every_day(),
      time_rule=time_rules.market_close(
          minutes=1))

def buy_low_sell_high(context, data):
  print(get_datetime())  # confirm 15:59
  # Get last 2 days of daily bars
  hist = data.history(
      context.security,
      bar_count=2,
      frequency='1d')
  close_yesterday = hist['close'].iloc[0]
  close_today = hist['close'].iloc[1]
  if close_today > close_yesterday:
    # Sell all: target 0% of portfolio
    order_target_percent(
        context.security, 0.0)
  else:
    # Buy all: target 100% of portfolio
    order_target_percent(
        context.security, 1.0)

CRITICAL ASSUMPTION: 3:59 PM ≈ Close

We assume: spot price at 15:59 ET is statistically equal to the true close price (16:00 ET).

How to validate: Compute historical difference between close price and 15:59 price → run a Student's t-test to check if the mean difference is statistically zero.

Why 15:59? We need yesterday's close to decide, but today's close isn't available until 16:00. We must act 1 minute before.

Order type: Market order at 15:59 to pursue immediate execution. It fits the model logic, but slippage remains part of the trade-off.

FIVE BACKTEST PITFALLS

1Fractional shares (most brokers don't support)
2Slippage (not simulated)
3Transaction cost (~0.5% in China, lower in US)
4Overfitting/underfitting (DOF analysis)
5Forward-looking bias (NOT in this model)

⚠️ IB DEFAULT COMMISSION: IBridgePy uses IB's real commission formula. Result: $1.42 for 285 shares at $350 = ~0.001%. Commission itself is tiny — slippage and adverse selection are the real costs.

🎯 KEY TAKEAWAYS

1. Overfitting/underfitting is determined by DATA PER DEGREE OF FREEDOM, not by backtest performance. Count your parameters (including any tuned via sweeping), divide total data points, and compare to the threshold: ≥1,000 for stock prices, 20-100 for general regression.

2. IBridgePy offers four backtesting methods of increasing speed: real-time IB requests (slowest/unstable), pre-ingested data, custom spot times, and simulated-by-daily-bars (fastest). Use daily bar simulation when minute data isn't available — Yahoo gives daily bars free.

3. Live trading demands handling scenarios backtests ignore: holidays, market early close (1 PM ET, 3-4 days/year), no-fill orders, margin errors, and IB server outages. Always use try/except + email notifications to keep the robot running without human intervention.

4. The buy-low-sell-high model uses only 3 DOF (~1667 data/DOF), runs unchanged for 8 years, and delivers Sharpe 0.8 vs SPY's 0.4. It's slightly underfitting — which is BETTER than overfitting. If you can't profit in your mathematical model, you won't profit in real markets.

THE MASTER FORMULA: Algo Trading = Right-fitting Model (data/DOF ≥ 1000 for stocks) + Robust Backtest (simulated daily bars, no look-ahead) + Live Error Handling (holidays, early close, margin, try/except) + Proper Deployment (local or cloud with backup plan).

THE ONE THING TO NEVER FORGET: "If you torture the data long enough, it will confess to anything." Before trusting ANY backtest result, count your degrees of freedom, divide by your data points, and check if the ratio meets the 1,000 threshold for financial data. A great-looking backtest with DOF/data < 20 is memorization, not prediction.
